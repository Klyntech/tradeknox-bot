# TradeKnox Deployment Guide

## Environment Variables

### Required (Bot won't start without these)

| Variable | Description | How to Get |
|----------|-------------|------------|
| `TELEGRAM_TOKEN` | Bot token from @BotFather | Message @BotFather on Telegram → /newbot → copy token |
| `PRIVATE_CHANNEL_ID` | Channel ID for signal delivery | Create channel → add bot as admin → forward message to @userinfobot → copy ID |

### Optional (Bot runs without them, but features disabled)

| Variable | Description | Default | How to Get |
|----------|-------------|---------|------------|
| `PUBLIC_CHANNEL_ID` | Channel ID for delayed signals (15 min) | Empty (disabled) | Same as private channel |
| `LICENSE_SECRET` | HMAC key for license signing | Auto-generated by Render | Leave empty — Render generates on first deploy |
| `STRIPE_SECRET_KEY` | Stripe API key | Empty (payments disabled) | Stripe Dashboard → Developers → API Keys |
| `STRIPE_WEBHOOK_SECRET` | Stripe webhook secret | Empty (webhooks disabled) | Stripe Dashboard → Developers → Webhooks → Signing Secret |
| `STRIPE_PRICE_PRO` | Stripe Price ID for Pro tier | Empty | Stripe Dashboard → Products → Create Price |
| `STRIPE_PRICE_VIP` | Stripe Price ID for VIP tier | Empty | Stripe Dashboard → Products → Create Price |
| `ACCOUNT_BALANCE` | Starting balance for position sizing | `10000` | Your trading account balance |
| `DATA_SOURCE` | Data provider | `twelvedata` | `twelvedata` (free 800/day), `yfinance` (fallback), `ccxt` (crypto) |
| `TWELVEDATA_API_KEY` | Twelve Data API key | Empty (falls back to yfinance) | [twelvedata.com](https://twelvedata.com) — free signup |

## Step-by-Step Deployment

### Prerequisites

- [ ] GitHub account
- [ ] Render account (free)
- [ ] Telegram account
- [ ] Phone access for @BotFather

### Step 1: Get Telegram Bot Token

1. Open Telegram, search for `@BotFather`
2. Send `/newbot`
3. Choose a name: `TradeKnox`
4. Choose a username: `TradeKnoxBot`
5. Copy the token (looks like `123456789:ABCdefGHIjklMNOpqrsTUVwxyz`)

### Step 2: Create Telegram Channels

**Private Channel (instant signals):**
1. Create a new channel (not group)
2. Name: `TradeKnox Signals`
3. Add your bot as admin
4. Forward a message from the channel to `@userinfobot`
5. Copy the channel ID (looks like `-1001234567890`)

**Public Channel (15 min delay, optional):**
1. Create another channel
2. Name: `TradeKnox Public`
3. Add your bot as admin
4. Get channel ID same way

### Step 3: Fork & Deploy

1. Fork `Klyntech/tradeknox-bot` to your GitHub
2. Go to [Render Dashboard](https://dashboard.render.com)
3. Click **New +** → **Web Service**
4. Connect your GitHub repo
5. Settings:
   - **Name:** `tradeknox-bot`
   - **Runtime:** Python
   - **Build Command:** `pip install -r requirements.txt`
   - **Start Command:** `python app.py`
   - **Plan:** Free
6. Click **Create Web Service**

### Step 4: Set Environment Variables

In Render Dashboard → your service → **Environment** tab:

1. Click **Add Environment Variable**
2. Add each variable:

```
TELEGRAM_TOKEN = your_bot_token_here
PRIVATE_CHANNEL_ID = -1001234567890
PUBLIC_CHANNEL_ID = -1009876543210  (optional)
TWELVEDATA_API_KEY = your_twelvedata_api_key
ACCOUNT_BALANCE = 10000
DATA_SOURCE = twelvedata
```

3. Click **Save Changes**
4. Render will auto-redeploy

### Step 5: Verify Deployment

1. Wait for deploy to complete (~2-3 minutes)
2. Check logs: Render Dashboard → your service → **Logs**
3. Look for: `Bot started polling...`
4. Open Telegram, find your bot, send `/start`
5. You should see a welcome message

### Step 6: Set Up UptimeRobot (Keep-Alive)

Render free tier sleeps after 15 min inactivity. UptimeRobot prevents this.

1. Go to [UptimeRobot](https://uptimerobot.com)
2. Create free account
3. Click **Add New Monitor**
4. Settings:
   - **Monitor Type:** HTTP(s)
   - **Friendly Name:** TradeKnox
   - **URL:** `https://tradeknox-bot.onrender.com/health`
   - **Monitoring Interval:** 5 minutes
5. Click **Create Monitor**
6. Done — bot stays awake 24/7

## Troubleshooting

### Bot doesn't start

**Check logs:** Render Dashboard → Logs

**Common errors:**
- `TELEGRAM_TOKEN is not set` → Set the token in environment variables
- `PRIVATE_CHANNEL_ID is not set` → Set the channel ID
- `ModuleNotFoundError` → Build command wrong, should be `pip install -r requirements.txt`

### Bot starts but no signals

**Check:**
1. Is the bot admin in the channel?
2. Is the channel ID correct (negative number)?
3. Are you in an allowed session (London/NY)?
4. Check logs for `No signal generated` — this is normal, means market conditions aren't right

### Bot stops after a while

**Cause:** Render free tier sleeps after 15 min inactivity

**Fix:** Set up UptimeRobot (Step 6 above)

### Webhook errors (Stripe)

**Check:**
1. Stripe webhook URL must be: `https://your-app.onrender.com/webhook`
2. Webhook secret must match `STRIPE_WEBHOOK_SECRET`
3. Enable the events: `checkout.session.completed`, `customer.subscription.updated`, `customer.subscription.deleted`

### Database errors

**Cause:** SQLite file corruption or permission issues

**Fix:**
- Render free tier has ephemeral storage — database resets on redeploy
- For production, consider PostgreSQL (Render free tier limitation)

## Local Testing

Before deploying, test locally:

```bash
# 1. Clone your fork
git clone https://github.com/YOUR_USERNAME/tradeknox-bot.git
cd tradeknox-bot

# 2. Install dependencies
pip install -r requirements.txt

# 3. Set environment variables
export TELEGRAM_TOKEN=your_token_here
export PRIVATE_CHANNEL_ID=-1001234567890

# 4. Run the bot
python app.py
```

**Note:** Local testing uses the same code as production. Database files (trades.db, licenses.db) are created locally.
